# Narrowing the Claim and Stress-Testing Leakage

Thank you for the careful reading. Your distinction between preserving psychometric information and claiming robust personality simulation helped us sharpen the claim. If accepted, in the camera-ready version we will state it more narrowly: the submitted pipeline shows high-fidelity transformation of psychometric profiles into generated life-story narratives and recovery from those narratives; it provides modest evidence of correspondence with real conversational behaviour; it does not by itself establish unconstrained or accurate simulation of a specific person.

**Real-conversation signal.** We agree that `r = 0.13-0.27` should be described as modest. We therefore ran targeted reference analyses using the same PARSEL conversations. Generated life-story features correlate with the same features coded from real conversations at mean `|r| = 0.147-0.180`. Participants' own HEXACO predicts those coded conversation features at mean `|r| = 0.094`, with a featurewise best-trait reference of `0.181`. PARSEL partner-perceived HEXACO correlates with self-report at mean `r = 0.157`, and independent LLM scorers recover HEXACO from the same short conversations at `r = 0.179-0.221`. If accepted, we will add these denominators and frame the result as modest behavioural triangulation, not strong behavioural simulation.

**Stage 1 audit.** The submitted system used Opus 4.6 to transform raw psychometrics into an immersive prompt. In response, we audited this step directly. We tested psychometric-only and psychometric-plus-biographical profile formats, both forbidding HEXACO names, explicit scores, percentiles, and instrumental scoring language, with post-generation forbidden-term checks. The resulting profiles decode at about `0.91-0.93` HEXACO before LSI generation, showing that this is a measured score-to-narrative transformation rather than an unexamined leakage source. A full-N prompt-writer comparison also found that Gemma 4 31B and Qwen 3.7 Max matched or exceeded the original Opus writer; Gemma is locally hostable, which improves reproducibility as proprietary endpoints change.

**Leakage checks.** Deleting every generated-narrative token that also appeared in the participant's profile removed about 20-26% profile/narrative lexical overlap, yet HEXACO recovery changed by only `-0.001` to `-0.020`. Deterministic scorecards made directly from source psychometrics, with no LLM profile writer, recovered HEXACO/Beyond-HEXACO at `0.654/0.515` for a compact scorecard and `0.728/0.584` for a glossary scorecard, both below the matched psychometric-only narrative profile at `0.761/0.618`. Thus direct numeric handoff carries signal, while the narrative transformation makes it more usable. We will present these as leakage and interpretability checks supporting the submitted recovery claim.

**Base models.** Kang et al.'s COLM 2025 work is directly relevant because it uses pretrained/base models for long interview-style backstories. Our setting differs: we transform measured psychometric profiles from the same 290 participants into leakage-controlled narrative prompts. Still, we ran full `N = 290` true base-model LSI generations with Gemma, Qwen, and OLMo base models. All exceeded participant-label-shuffle controls, so the recovery effect is not limited to chat-model compliance. For this measured-profile transformation task, instruction-tuned models remained stronger and more controllable.

**Dark-trait slice.** We also tested whether safety tuning might sanitise dark-trait-adjacent profiles. On a 50-PID dark/bright slice, base models did not show a special advantage. To match the base-model setup in Kang et al. (arXiv:2504.11673), we ran `mistralai/Mistral-Small-24B-Base-2501`; it recovered HEXACO at `r = 0.556`, dark-axis aggregate at `r = 0.607`, and PPTS total at `r = 0.657`, below regular Qwen/Gemma. Less safety-filtered refusal-ablation variants also showed no meaningful advantage. If accepted, we will report this as a targeted base-model and refusal-ablation control.

**Human check.** Cross-provider LLM evaluation does not eliminate shared model bias, so we ran a new human-reader pilot analysing only AI-generated materials. Twenty-five native-English Prolific raters each completed 25 four-option matches using brief snippets of profile information and generated LSI themes, passing bot/LLM screening and attention checks. Accuracy was significantly above chance, `59.5%` versus `25%`, `t(24)=8.24`, one-tailed `p<1e-8`. The limited information shown to raters makes this a conservative human check. We will also stop calling human test-retest reliability a ceiling; if accepted, we will describe it as a published test-retest reference and separately report the direct profile-decode reference.
